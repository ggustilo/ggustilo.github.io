Classes vs Ids

September 27, 2015

classes

ids

css

best practices

In CSS, one of the best ways to style your content is to make use of classes and ids. By incorporating a class or id attribute in a tag, you can call certain styles associated with that class selector onto your content. For example, you might call a class of "quote" onto a paragraph - and thus designate it for indented italic styles, perhaps in a different color. Classes are to be used more than once - creating a class of elements which are all styled the same (ex. all the quotes in your document). Ids are used only once - when one element needs unique styles. For example, maybe you have a particular banner image that is only used once, on the homepage of your document, and it needs a certain opacity and size. You might give it a unique id of "homepage-banner."

However, CSS best practices are constantly evolving, and one new standard suggests reserving the use of ids for Javascript handles, and using only classes for CSS. This is because Javascript also needs to use some kind of selector to work - either the element tags themselves, or a class, or ids. The benefit of using ids exclusively for Javascript is that you can quickly scan an html file and see where Javascript is incorporated. Although I usually follow this practice, it doesn't always fit the bill. For example, what if you have a piece of Javascript code that needs to work on a class of elements?

As Javascript - and perhaps other technologies - are more commonly included in web applications and make use of CSS selectors, perhaps other tricks will evolve to more cleanly separate functionality in the document.
